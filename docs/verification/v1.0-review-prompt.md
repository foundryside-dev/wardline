# v1.0 Pre-Release Review Prompt

Use this prompt to dispatch an agent for a top-to-bottom conformance review of the wardline Python implementation against the Wardline Framework Specification v0.2.0 and Python Binding v0.2.0.

## Prompt

```
You are conducting a pre-release conformance review of the wardline Python implementation (v1.0.0) against the Wardline Framework Specification.

## Objective

Produce a structured conformance verdict covering every normative section of the spec. For each section, determine whether the implementation satisfies all MUST/SHALL requirements, identify any gaps or deviations, and assess whether deviations are narrowing (acceptable) or widening (non-conformant).

Decorators: src/wardline/decorators/ (__init__.py, _base.py, _introspect.py, authority.py, boundaries.py, + 14 more)

CLI: src/wardline/cli/ (scan.py, main.py, + 8 command modules)
Corpus: corpus/ (252 specimens, corpus_manifest.json)
Self-hosting: wardline.yaml, wardline.exceptions.json, wardline.fingerprint.json

## Prior Assessment

An internal assessment exists at docs/requirements/spec-fitness/assessment-2026-03-29.md showing 101 pass / 5 partial / 0 fail across 106 requirements. Your job is to independently verify this — do not trust the self-assessment. Check the code.

## Review Structure

For each spec section, produce:

1. **Section ID and title**
2. **MUST requirements extracted** — list every normative MUST/SHALL statement
3. **Implementation evidence** — file:line references proving each requirement is met
4. **Verdict** — PASS / PARTIAL / FAIL with justification
5. **Deviations** — any narrowing deviations (acceptable) or widening deviations (non-conformant)
6. **Risks** — anything that technically passes but feels fragile or under-tested

## Special Attention Areas

- §4 Authority Tier Model: verify TAINT_TO_TIER is complete, taint_join algebra is correct
- §5 Enforcement Spec: verify skip-promotion rejection, serialisation sinks, restoration evidence bounds
- §6 Annotation Vocabulary: verify all 17 groups present in registry.py
- §7 Pattern Rules: verify all 9 rules implemented with correct detection patterns
- §8 Enforcement Layers: verify 3-level taint propagation, SARIF determinism
- §9 Governance Model: verify control law, exception lifecycle, fingerprint tracking
- §10 Verification Properties: verify corpus oracle properties, precision/recall floors
- §13 Manifest Format: verify schema validation, overlay merging, narrow-only invariant
- §14 Conformance: verify Lite profile checklist, assessment procedure
- Python Binding §A: verify all SARIF properties, exit codes, tier-aware error handling

## Output Format

Produce a single markdown document with:
1. Executive summary (pass/partial/fail counts, overall verdict)
2. Per-section detailed review (as structured above)
3. Risk register (anything that could bite an adopter)
4. Final recommendation (ship / ship with caveats / hold)
```
