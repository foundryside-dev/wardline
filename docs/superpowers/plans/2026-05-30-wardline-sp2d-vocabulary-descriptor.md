# SP2d — NG-25 Vocabulary Descriptor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Export the trust-decorator `REGISTRY` as a plain, version-stamped NG-25 vocabulary descriptor — `build_vocabulary_descriptor()`, a committed `src/wardline/core/vocabulary.yaml` shipped in the wheel, and a `wardline vocab` CLI subcommand — so federation consumers (Clarion) can *read* the vocabulary instead of *importing* it.

**Architecture:** `core/descriptor.py` builds a `dict` envelope `{"version": REGISTRY_VERSION, "entries": [...]}` directly from `REGISTRY`. Each entry mirrors `REGISTRY`'s three fields — `canonical_name`, `group`, and `attrs` (the attr-name → taint-type mapping, serialized as `{name: type.__name__}`). The §2 `FunctionTaint` mapping is *parametric* (computed by `DecoratorTaintSourceProvider` from call-site `to_level`/`level` kwargs) and provider-owned, so it is deliberately **out of scope** for the descriptor — the descriptor round-trips `REGISTRY`, not the provider. The committed `vocabulary.yaml` is a derived snapshot of `build_vocabulary_descriptor()`; a drift test (`yaml.safe_load(file) == build_vocabulary_descriptor()`) keeps it from going stale, and the regeneration command is a single documented one-liner. No HMAC / signing / governance — a plain versioned YAML export.

**Tech Stack:** Python 3.12, `click` (CLI), `PyYAML` (`yaml.safe_dump`/`safe_load`), `importlib.resources` (wheel-shipped data, mirroring `stdlib_taint.py`), `hatchling` (wheel `force-include` for the data file).

```python
# src/wardline/core/descriptor.py
"""NG-25 vocabulary descriptor — the federation-clean *read-instead-of-import*
export of the trust-decorator ``REGISTRY``.

``build_vocabulary_descriptor()`` returns a plain, JSON/YAML-serializable dict
``{"version": REGISTRY_VERSION, "entries": [...]}``. Each entry mirrors REGISTRY's
three fields — ``canonical_name``, ``group``, and ``attrs`` (the attr-name →
taint-type mapping, serialized as ``{name: type.__name__}``). The §2 FunctionTaint
mapping is parametric and provider-owned (``DecoratorTaintSourceProvider`` reads
``to_level``/``level`` from each call site), so it is deliberately out of scope:
the descriptor round-trips REGISTRY, not the provider.

No HMAC / signing / governance — a plain versioned export. The committed
``vocabulary.yaml`` (shipped in the wheel) is a derived snapshot of this function;
a drift test keeps it current.
"""

from __future__ import annotations

from typing import Any

import yaml

from wardline.core.registry import REGISTRY, REGISTRY_VERSION


def build_vocabulary_descriptor() -> dict[str, Any]:
    """Export REGISTRY as the NG-25 descriptor dict (entries in REGISTRY order)."""
    return {
        "version": REGISTRY_VERSION,
        "entries": [
            {
                "canonical_name": entry.canonical_name,
                "group": entry.group,
                "attrs": {name: typ.__name__ for name, typ in entry.attrs.items()},
            }
            for entry in REGISTRY.values()
        ],
    }


def descriptor_to_yaml() -> str:
    """Serialize the descriptor to deterministic YAML (key order preserved)."""
    return yaml.safe_dump(
        build_vocabulary_descriptor(),
        sort_keys=False,
        default_flow_style=False,
    )
```

```python
    # If this fails, REGISTRY changed and the file is stale — regenerate with:
    #   .venv/bin/wardline vocab > src/wardline/core/vocabulary.yaml

        files("wardline.core").joinpath("vocabulary.yaml").read_text(encoding="utf-8")
```
